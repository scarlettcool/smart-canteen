# 智慧食堂管理后台 PRD (Admin Portal)

## 1. 角色权限 (RBAC)
- **系统管理员**: 全量权限。
- **财务审计**: 仅查看流水、充值记录、导出对账单。
- **后勤管理**: 审核员工注册、处理投诉申诉。
- **食堂店长**: 管理菜品库存、查看实时订单、叫号处理。

## 2. 核心模块与闭环逻辑

### 2.1 员工审核流 (Register Loop)
- **页面内容**: 待审核列表（姓名、工号、照片、部门）。
- **操作**: [通过] [驳回+填写原因]。
- **数据反馈**: 审核结果即时更新 `users.reg_status`，用户端 H5 刷新状态页可见。

### 2.2 菜品与库存 (Menu Loop)
- **页面内容**: 菜品网格管理。
- **字段**: 名称、价格、所属档口、剩余库存、上/下架开关。
- **数据反馈**: 修改后用户端 `CanteenMenu` 实时同步。

### 2.3 预约监控 (Reservation Loop)
- **页面内容**: 各食堂/时段预约人数统计。
- **操作**: 设定每日预约上限（Capacity Control）。
- **数据反馈**: 当人数达到上限，用户端 `Reservation` 对应日期变为 `full`。

### 2.4 订单与对账 (Wallet Loop)
- **页面内容**: 全量支付流水查询。
- **字段**: 订单 ID、员工姓名、消费时间、金额、支付状态。
- **审计**: 支持导出 CSV 报表。

### 2.5 申诉处理 (Appeal Loop)
- **页面内容**: 投诉/申诉待办。
- **操作**: [回复建议] [撤销违约记录]。
- **数据反馈**: 撤销后用户信用恢复，可重新预约。

## 3. 状态转换约束
- **订单退款**: 必须由财务角色手动触发，且资金原路退回 `users.balance` 并生成 `wallet_logs`。
- **食堂关闭**: 一键关闭食堂后，前端 `CanteenSelect` 对应项置灰并标记 `closed`。
