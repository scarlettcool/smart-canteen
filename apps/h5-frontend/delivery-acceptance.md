# 智慧食堂用户端 (V3.1) 交付验收说明

## 1. 交付范围 (Scope)
本次交付包含完整的移动端 H5 交互原型及核心业务逻辑模拟：
- **核心模块**: 个人中心、实名认证审核、钱包充值/流水、点餐支付、就餐预约/取消、理发取号、优惠券激活。
- **交互规范**: 全量页面转场动画、加载/空状态/错误处理、多主题定制。
- **技术栈**: React 19 + TypeScript + Tailwind CSS + Lucide Icons。

## 2. 状态确认 (Truth vs Mock)
| 模块 | 实现程度 | 验收说明 |
| :--- | :--- | :--- |
| **UI/UX** | 100% 真实 | 像素级还原，适配主流移动端刘海屏。 |
| **数据存储** | 模拟持久化 | 使用 `localStorage` 模拟后端存储，支持刷新后状态保留。 |
| **API 调用** | 异步模拟 | `ApiService` 封装了 800ms 延迟及错误注入，模拟真实请求体验。 |
| **后端逻辑** | 逻辑闭环 | 余额扣除、预约状态锁定、流水生成均由 `ApiService` 逻辑计算完成。 |

## 3. 验收证据链 (Acceptance Evidence)
验收人可按以下流程验证“可验收证据链”：
1.  **实名拦截**: 首次进入，点击“堂食点餐”应弹出实名拦截 -> 前往“注册” -> 提交申请 -> 模拟通过 -> 获得权限。
2.  **金额同步 (SOT)**: 在“我的钱包”充值 300 元 -> 返回首页验证余额更新 -> 前往菜单点餐 -> 支付后验证余额扣除且生成流水。
3.  **状态机闭环**: 进行一次“就餐预约” -> 在“记录”中查看待处理预约 -> 进入详情点击“取消” -> 验证状态变更为“已取消”。
4.  **异常捕获**: 将余额消耗至 0 -> 尝试再次点餐 -> 验证触发 `E2001 (余额不足)` 错误弹窗及 [去充值] CTA 引导。

## 4. 后续计划
- **真实对接**: 替换 `api/service.ts` 中的 Mock 逻辑为 `fetch` 调用真实后端接口。
- **推送集成**: 引入 WebSocket 或 SSE 实现预约即将过期的系统实时提醒。
- **离线能力**: 配置 PWA Service Worker 提升弱网环境下的加载速度。

---
**交付结论**: 该版本已具备完整的业务逻辑闭环与交付证据链，可作为后端开发与最终用户验收的基准。
