
# E2E 业务闭环自测矩阵

| 模块 | 场景 | 预期行为 | 测试结果 |
| :--- | :--- | :--- | :--- |
| **基础** | 全站刷新 | `localStorage` 中的余额、认证状态、主题应保持不变 | ✅ |
| **认证** | 未认证支付 | 点击“堂食点餐”应准确跳转 `/register` 且不可绕过 | ✅ |
| **钱包** | 充值路径 | 选择 100 元 -> 支付 -> 结果页 -> 首页余额更新 (+100) | ✅ |
| **钱包** | 余额不足 | 余额为 0 时去下单 -> 触发 `E001` -> 弹窗提示并引导去充值 | ✅ |
| **预约** | 取消流程 | 待就餐 -> 取消 -> 状态变为“已取消” -> 列表页同步更新 | ✅ |
| **预约** | 批量提交 | 选择 3 天 -> 提交 -> 成功后验证列表出现 3 条记录 | ✅ |
| **点餐** | 重复点击 | 狂点“结算”按钮 -> 仅触发一次 API 且中间状态按钮禁用 | ✅ |
| **全局** | 退出登录 | 点击退出 -> Clear -> 再次访问 `/orders` 应被 Auth 拦截 | ✅ |
