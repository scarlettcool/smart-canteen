# Sprint 1 页面字段与交互契约

## 1. 人员档案 (Archive)
- **核心实体**: `UserArchive`
- **交互逻辑**: 
  - 修改工号需触发 409 冲突检查。
  - 删除操作必须为“软删除”，且需二次确认。
  - 余额字段只读，由 Sprint 2 账户管理模块修改。

## 2. 注册审批 (Approval)
- **核心实体**: `RegRecord`
- **交互逻辑**:
  - 批量通过：选中 ID 数组发送至 `/approvals/batch`。
  - 驳回：`rejectReason` 强制校验，不可为空。

## 3. 组织架构 (Org)
- **核心实体**: `DeptNode`
- **交互逻辑**:
  - 树形结构实时刷新。
  - 部门主管变更需记录审计日志。

## 4. 编号规则 (ID Rule)
- **交互逻辑**:
  - 输入 Prefix/Serial 时，前端需实时合成 Preview 字符串。
