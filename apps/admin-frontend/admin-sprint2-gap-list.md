# Sprint 2 消费管理 Gap List (Question List for Antigravity)

## 13. 账户管理
| 项目 | 内容 |
| :--- | :--- |
| 功能编号 | 13 |
| 菜单路径 | 消费管理 > 账户管理 |
| 功能点 | 余额调整 (调账) |
| 页面状态 | 已完成 |
| 前端现状 | 调账弹窗、金额校验、原因必填、RequestId 生成逻辑已实现。 |

| 是否影响联调 | 是 |
| 是否阻塞 Sprint3 | 否 |
| 处理建议 | Antigravity 后端实现账本服务及幂等拦截。 |

## 15. 退款审核
| 项目 | 内容 |
| :--- | :--- |
| 功能编号 | 15 |
| 菜单路径 | 消费管理 > 交易管理 > 退款审核 |
| 功能点 | 通过并执行退款 |
| 页面状态 | 占位 |
| 前端现状 | 审批状态机 UI（通过/驳回）已完成，点击“通过”后会弹出“执行原路退款”提示。 |
| 缺失点类型 | 第三方支付 |
| 依赖说明 | 后端需对接 `WechatPay/Alipay SDK` 的退款接口，处理异步通知结果，并在退款成功后反写系统流水状态。 |
| 是否影响联调 | 是 |
| 是否阻塞 Sprint3 | 否 |
| 处理建议 | 后端需实现外部支付网关对冲逻辑，前端仅需根据状态字段展示结果。 |
