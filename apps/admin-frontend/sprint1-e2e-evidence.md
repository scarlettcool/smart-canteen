# Sprint 1 E2E 验收路径指南 (1-12)

### T01: 人员档案全闭环 (1)
*   **入口**: `/people/archive/list`
*   **验收步骤**:
    1. 点击右上角 [新增档案]。
    2. 输入重复工号，点击保存 -> 预期：弹出 "工号冲突 (40901)" 提示。
    3. 修改为唯一工号，点击保存 -> 预期：Toast 成功，列表自动刷新。
    4. 勾选一行，点击删除 -> 预期：弹出二次确认，状态变为“已离职”。
*   **关联接口**: `POST /people/archives`, `DELETE /people/archives/:id`

### T02: 组织架构动态调整 (2)
*   **入口**: `/people/archive/org`
*   **验收步骤**:
    1. 选中左侧树 [技术开发中心]。
    2. 点击 [编辑]，修改名称，保存 -> 预期：左侧树同步更新。
    3. 点击 [变更历史] -> 预期：右侧滑出日志，显示刚才的修改记录。
*   **关联接口**: `GET /org/tree`, `PUT /org/departments`

### T03: 注册审批批量流转 (3)
*   **入口**: `/people/archive/approval`
*   **验收步骤**:
    1. 勾选多条“待审核”记录。
    2. 点击 [批量通过] -> 预期：Loading 覆盖表格，完成后勾选行消失。
    3. 点击单行 [驳回] -> 预期：未填原因无法提交。
*   **关联接口**: `POST /approvals/batch`

### T04: 工号规则实时预览 (11)
*   **入口**: `/people/option/id-rule`
*   **验收步骤**:
    1. 修改 [前缀 (Prefix)] 为 "EMP"。
    2. 观察下方 [实时预览] 区域 -> 预期：瞬间变为 "EMP-2024-001"。
*   **关联接口**: `POST /rules/id-generate/preview`

### T05: 黑名单管控 (12)
*   **入口**: `/people/option/blacklist`
*   **验收步骤**:
    1. 切换至 [命中记录] 页签 -> 预期：显示拦截日志。
    2. 在 [受限名单] 中点击 [解除限制] -> 预期：弹出二次确认。
*   **关联接口**: `GET /blacklist`, `PUT /blacklist/:id/lift`
