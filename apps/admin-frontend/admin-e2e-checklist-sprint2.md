# Sprint 2 消费管理 E2E 验收自测 Checklist

## 第一阶段：账户与交易 (P0 核心账务)
- [ ] **调账测试**: 点击 [调账] -> 输入重复 requestId -> 预期: 第二次被幂等拦截。
- [ ] **调账反馈**: 输入金额 10.00 -> 确认 -> 余额实时更新为 余额+10.00 -> 刷新页面数据一致。
- [ ] **冲正测试**: 寻找一笔昨天流水 -> 点击冲正 -> 预期: 若账期已关应提示失败，否则成功返还资金。
- [ ] **冻结拦截**: 冻结某用户 -> 在消费流水页模拟该用户支付 -> 预期: 状态显示 "Rejected - Frozen"。

## 第二阶段：退款与申诉 (P1 业务流转)
- [ ] **退款审批**: 提交申请 -> 财务主管进入 [退款审核] -> 审核驳回 -> 预期: 申请人收到微信驳回消息。
- [ ] **失约补偿**: 处理申诉 -> 点击 [通过并返还罚金] -> 预期: 自动产生一条 `recharge` 类型流水。

## 第三阶段：报表中心 (P1 数据一致性)
- [ ] **筛选器联动**: 改变日期范围 -> 点击查询 -> 列表首行日期必须在范围内。
- [ ] **口径校验**: 点击 [口径定义] 抽屉 -> 核对计算公式与下方表格表头是否一致。
- [ ] **异步导出**: 点击导出 -> 刷新页面 -> 进入下载中心 -> 下载文件并核对内容格式。

## 第四阶段：配置与硬件 (P2 运维控制)
- [ ] **限额变更**: 修改单次限额 -> 保存 -> 重新打开该页 -> 预期: 修改已持久化。
- [ ] **指令下发**: 找到一台离线设备 -> 点击重启 -> 预期: 提示 "指令排队中"，状态变为 "Offline-Pending"。
